[#xtheadmemidx-insns-lwuib,reftext=Load indexed unsigned word, increment-before]
==== th.lwuib

Synopsis::
Load indexed unsigned word, increment address before loading.

Mnemonic::
th.lwuib _rd_, (_rs1_), _imm5_, _imm2_

Encoding::
[wavedrom, , svg]
....
{reg:[
    { bits:  7, name: 0xb, attr: ['custom-0, 32 bit'] },
    { bits:  5, name: 'rd' },
    { bits:  3, name: 0x4, attr: ['Mem-Load'] },
    { bits:  5, name: 'rs1' },
    { bits:  5, name: 'imm5' },
    { bits:  2, name: 'imm2' },
    { bits:  5, name: 0x19 },
]}
....

Description::
This instruction increments the value in _rs1_ by (sign_extend(_imm5_) << _imm2_) and writes the result back to _rs1_.
After the increment of _rs1_, this instruction loads a zero extended 32-bit value into the GP register _rd_ from the (incremented) address _rs1_.

The values of _rd_ and _rs1_ must not be the same.

Operation::
[source,sail]
--
// illegal instruction exceptions
if (mxstatus.theadisaee != 1)
{
  <raise illegal instruction exception>
}

// execute instruction
rs1 := rs1 + (sign_extend(imm5) << imm2)
rd := zero_extend(mem[rs1+3:rs1])
--

Permission::
This instruction is available in `M` mode, `S` mode, and `U` mode.

Exceptions::
This instruction triggers the same exceptions that a corresponding `LWU` instruction would trigger.

Availability::
The instruction is only available if `mxstatus.theadisaee` is set to `1`.
Otherwise this instruction will trigger an illegal instruction exception.
